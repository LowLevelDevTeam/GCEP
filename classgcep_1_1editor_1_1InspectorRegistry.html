Detailed Description

Singleton that maps each component typeID to a draw function.

Components are drawn in registration order (insertion order preserved). Default registration uses Boost.PFR to iterate fields automatically. A custom lambda can override the default for components that need specific widgets (combos, sliders with range, etc.).

Each component is rendered inside a CollapsingHeader, open by default.
