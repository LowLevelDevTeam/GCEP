Detailed Description

Owns all per-frame-in-flight Vulkan objects for a single frame slot.

With MAX_FRAMES_IN_FLIGHT = 2 the CPU can record frame N+1 while the GPU is still executing frame N, without ever stalling on the second frame.

Member roles
  • commandPool - Created with eTransient; reset wholesale each frame via vkResetCommandPool, which is faster than resetting individual buffers.
  • commandBuffer - Primary buffer allocated from the pool; re-recorded every frame.
  • commandList - Typed wrapper around commandBuffer.
  • inFlightFence - Pre-signaled at construction so frame 0 does not stall before the GPU has done any work. Waited on and reset in waitAndReset() before each recording.
  • presentCompleteSemaphore - Signaled by vkAcquireNextImageKHR when the swapchain image is safe to write. The GPU waits at eColorAttachmentOutput before writing color.
Destruction order
Member declaration order is also the reverse-destruction order enforced by C++. commandList -> commandBuffer -> commandPool -> inFlightFence -> presentCompleteSemaphore.
Thread safety
Not thread-safe. Owned and used exclusively by the render thread.

Constructs and initialises all per-frame Vulkan objects.

Allocates a transient command pool for queueFamily, allocates a single primary command buffer from it, creates a pre-signaled in-flight fence, and creates the acquire semaphore. The VulkanCommandList is constructed last, wrapping the allocated command buffer.

Parameters
deviceThe logical device used to create all child objects. Must remain valid for the lifetime of this struct.
queueFamilyIndex of the graphics/present queue family. All objects are created targeting this family.

◆ waitAndReset()

void gcep::rhi::vulkan::VulkanFrameContext::waitAndReset ( const vk::raii::Device &  device)

Blocks the CPU until the GPU finishes all work submitted with this frame slot's fence, then resets the fence and command pool.

This is the "begin of frame" CPU stall. Calling vkResetCommandPool in one shot is faster than resetting individual command buffers because the driver can reclaim all allocator memory at once.

Parameters
deviceThe logical device; must be the same one used at construction.
Exceptions
std::runtime_errorif vkWaitForFences returns any result other than VK_SUCCESS (e.g. VK_ERROR_DEVICE_LOST).
