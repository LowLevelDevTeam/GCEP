Detailed Description

RAII manager for the Dear ImGui context and its Vulkan + GLFW backends.

ImGuiManager centralises the three-step ImGui lifecycle — initialisation, per-frame begin/end, and shutdown — so that the rest of the editor code only calls the high-level helpers and never touches the raw backend functions directly.

Typical usage
UI::ImGuiManager imguiManager;
imguiManager.init(glfwWindow, vulkanInitInfo);
while (running)
{
imguiManager.beginFrame();
// ... ImGui calls ...
imguiManager.endFrame();
}
imguiManager.shutdown();

void gcep::UI::ImGuiManager::beginFrame ( )

Begins a new Dear ImGui frame.

Calls ImGui_ImplVulkan_NewFrame(), ImGui_ImplGlfw_NewFrame(), and ImGui::NewFrame() in that order. Must be called once per render loop iteration before any ImGui::* draw calls.

void gcep::UI::ImGuiManager::init ( GLFWwindow *  window,
ImGui_ImplVulkan_InitInfo  initInfo 
)

Initialises the Dear ImGui context and both platform/renderer backends.

Steps performed:

  1. Creates the ImGui context and configures the ImGuiIO flags (docking, viewports).
  2. Calls ImGui_ImplGlfw_InitForVulkan() with window.
  3. Calls ImGui_ImplVulkan_Init() with initInfo.
  4. Uploads the font atlas to the GPU.
Parameters
windowGLFW window to bind the GLFW backend to. Must outlive this object.
initInfoVulkan backend initialisation parameters (instance, device, queue, etc.).
