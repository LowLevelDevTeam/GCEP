Detailed Description

template<ComponentConcept T>
class gcep::ECS::ComponentPool< T >

Component pool implementation based on a Sparse Set data structure.

Manages storage and lifecycle for components of type T. Uses three synchronized structures for optimal performance:

  • A Paged Sparse Array for O(1) direct access via EntityID.
  • A Dense Array for grouping active entities to allow fast iteration.
  • A contiguous data array to maximize CPU cache efficiency.
    Template Parameters
    TThe type of component stored in this pool.

Constructs and attaches a component of type T in-place using provided arguments.

Uses variadic templates and perfect forwarding to invoke the constructor of T directly within the pool's storage. This is the most efficient way to initialize components with specific data, as it eliminates temporary copies.
