Detailed Description

Owns a VkSwapchainKHR, all swapchain VkImage handles, and their VkImageViews.

Ownership model
Swapchain images (VkImage) are owned by the driver; this class stores them as plain vk::Image (non-owning). Image views (VkImageView) are owned here via vk::raii::ImageView and destroyed when this object is destroyed.
Lifecycle
Constructed by VulkanDevice during initSwapchain(). On window resize, VulkanDevice::recreateSwapchain() destroys the old instance (via unique_ptr::reset()) before constructing a new one, preventing ErrorNativeWindowInUseKHR.
Thread safety
Not thread-safe. Used exclusively on the render thread.

Constructs the swapchain, creates all swapchain images, and creates image views.

Selects the optimal surface format (prefers B8G8R8A8_SRGB / eSrgbNonlinear), present mode (prefers Mailbox when !desc.vsync, falls back to FIFO), and extent (clamps desc.width / desc.height to the surface capabilities). Requests a minimum of 3 images, clamped to the surface's maxImageCount.

Parameters
deviceThe logical device used to create the swapchain and image views.
physicalDeviceUsed to query surface capabilities, formats, and present modes.
surfaceThe VkSurfaceKHR to create the swapchain for.
descCreation parameters (window size, vsync preference).
Exceptions
std::runtime_errorif the surface reports no formats or present modes.

std::pair< vk::Result, uint32_t > gcep::rhi::vulkan::VulkanSwapchain::acquireNextImage ( const vk::raii::Semaphore & signalSemaphore) const
nodiscard

Acquires the next presentable swapchain image.

Calls vkAcquireNextImageKHR with an infinite timeout. The semaphore is signaled by the presentation engine when the image is safe to write.

Parameters
signalSemaphoreBinary semaphore to signal when the image becomes available. Must be unsignaled when this call is made; use one semaphore per frame slot (not per image) to guarantee this.
Returns
A vk::ResultValue containing:
  • eSuccess - image acquired normally; value is the image index.
  • eSuboptimalKHR - still usable; caller may recreate swapchain after present.
  • eErrorOutOfDateKHR - swapchain is stale; caller must recreate before presenting.
