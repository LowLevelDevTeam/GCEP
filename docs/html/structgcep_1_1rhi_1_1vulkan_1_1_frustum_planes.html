Detailed Description

Frustum plane data passed as a push constant to the culling compute shader.

Populated each frame by VulkanRHI::extractFrustum() using the Gribb-Hartmann method applied to the combined proj * view matrix. Each plane is stored as a normalized vec4 where xyz is the inward-facing normal and w is the signed distance from the origin, such that dot(n, p) + d >= 0 means the point p is inside (or on) the plane.

Note
The three float padding fields are required so the total struct size is a multiple of 16 bytes, satisfying the Vulkan push constant alignment rules.
