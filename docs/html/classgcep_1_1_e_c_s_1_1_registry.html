Detailed Description

Central Manager for the ECS (Entity Component System).

The Registry handles the lifecycle of entities, paged allocation of signatures, and maintains the link between entities and their respective components.

EntityID gcep::ECS::Registry::createEntity ( )

Creates a new entity.

Uses the EntityIDGenerator to allocate a new entity ID. If a destroyed entity slot is available in the free list, that ID is reused with an incremented version number. Otherwise, a new slot is allocated from the entity pool.

Returns

◆ destroyEntity()

void gcep::ECS::Registry::destroyEntity ( EntityID entity)
inline

Marks an entity for deferred destruction.

The entity is queued for removal and its ID is invalidated through version checking by the EntityIDGenerator. The entity remains logically valid until the next call to update(), where it is permanently removed and its slot recycled.

Note
The entity remains valid until the next call to update().
Parameters
entityThe identifier of the entity to be removed.
