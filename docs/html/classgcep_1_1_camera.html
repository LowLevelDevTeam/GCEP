Detailed Description

First-person editor camera that maps keyboard/mouse events to view and projection matrices consumed by the RHI.

Camera polls an InputSystem each frame to translate WASD / QE keyboard input into positional movement and drag-based mouse input into yaw/pitch rotation. The resulting matrices are packed into a rhi::vulkan::UniformBufferObject and returned by update().

Coordinate system
  • The world-up axis is (0, 0, 1) (Z-up).
  • The camera's local up is (0, 1, 0).
  • Yaw is measured from the negative-X axis, pitch from the XY plane.
Typical usage
Camera camera(&inputSystem, &window);
// each frame:
auto ubo = camera.update(aspectRatio, settings.cameraSpeed);
rhi->updateCameraUBO(ubo);

rhi::vulkan::UniformBufferObject gcep::Camera::update ( float aspect,
float camSpeed )
nodiscard

Advances the camera simulation by one frame and returns the updated UBO.

Processes movement keys (WASD / QE) scaled by camSpeed, updates yaw/pitch from right-mouse-button drag, then recomputes front, right, and velocity. Writes the new view/projection matrices and the camera position into ubo before returning it.

Parameters
aspectViewport width divided by height.
camSpeedMovement speed in world units per second.
Returns
The updated UniformBufferObject ready to be uploaded to the GPU.
