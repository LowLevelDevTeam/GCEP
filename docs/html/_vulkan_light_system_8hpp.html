Detailed Description

CPU-side light data types, GPU-mapped buffer structs, and the LightSystem class that owns all Vulkan resources for point and spot lights.

Architecture overview
The light system is a self-contained subsystem that plugs directly into VulkanRHI. It owns:
  • A persistently-mapped, host-visible/coherent SSBO containing packed GPULight entries (one per live light, up to MAX_LIGHTS).
  • A host-visible/coherent metadata UBO (LightMetaUBO) that carries the live light count so the lighting compute shader can early-exit.
  • A compute pipeline (Compute_Lighting.spv, entry compMain) that reads the G-buffer-like offscreen color + depth produced by the scene pass, the light SSBO, and writes a shaded output into an RGBA16F storage image. The result is fed into the TAA resolve stage as the new "current color".
  • All descriptor set layouts, pools, and sets needed for the above.
Descriptor set layout (set 3 - lighting compute)
  • Binding 0 eStorageBuffer - GPULight SSBO (read-only)
  • Binding 1 eUniformBuffer - LightMetaUBO (read-only)
  • Binding 2 eCombinedImageSampler - offscreen color (read)
  • Binding 3 eCombinedImageSampler - offscreen depth (read)
  • Binding 4 eStorageImage - lit output (write)
Push constants (LightingPushConstants, compute stage)
  • invViewProj - inverse view-projection for depth-buffer ray reconstruction
  • cameraPos - world-space eye position for specular computation
  • screenSize - pixel dimensions of the offscreen render target
Integration in VulkanRHI::recordOffscreenCommandBuffer()
Insert recordLightingPass() after endRendering() and before recordTAAPass(). The lighting pass reads the scene color and depth as sampled images and writes the lit result into m_litImage (storage), then transitions it to eShaderReadOnlyOptimal for TAA.
Thread safety
All public methods must be called from the render thread. No internal synchronisation is provided.

Identifies the light model encoded in a GPULight entry.

The value is stored in GPULight::type and is tested by the Slang shader to branch between the point-light and spot-light attenuation models.

Enumerator
Point 

Isotropic point light with inverse-square attenuation.

Spot 

Cone-shaped spot light with inner/outer cutoff angles.
