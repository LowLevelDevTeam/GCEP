Detailed Description

Manages the lifetime and GPU resources of a 2D Vulkan texture.

Handles the full pipeline from loading an image file off disk to registering it in the global bindless descriptor array: staging upload, optional mip-chain generation, image view creation, sampler configuration, and bindless slot registration.

Typical usage:

VulkanTexture tex;
tex.loadTexture(rhi, "assets/albedo.png", true);
uint32_t slot = tex.getBindlessIndex(); // pass to GPUMeshData::textureIndex

const std::filesystem::path & filepath,
bool mipmaps = true )

Loads an image from disk and registers it in the global bindless array.

Internally allocates a host-visible staging buffer, copies pixel data into it, transitions the image layout to eTransferDstOptimal, submits a buffer-to-image copy, and — when mipmaps is true — generates a full mip chain via generateMipmaps(). The final layout is eShaderReadOnlyOptimal. Finally calls registerBindless() to write the image view + sampler into the bindless descriptor set at an atomically allocated slot.

Parameters
instanceNon-owning pointer to the owning VulkanRHI context.
filepathPath to the source image (any format supported by stb_image).
mipmapsWhen true, generates a full mip chain down to 1×1. When false, the texture has a single mip level.
Exceptions
std::runtime_errorif the file cannot be opened or decoded.

◆ setLodLevel()

void gcep::rhi::vulkan::VulkanTexture::setLodLevel ( float lodLevel)

Dynamically adjusts the minimum LOD bias of the sampler.

Waits for the device to be idle, recreates the VkSampler with lodLevel as the minLod clamp, and re-registers the texture in the bindless set at its existing slot. Useful for visualising individual mip levels or implementing LOD-fade effects at runtime.

Only has an effect if the texture was loaded with mipmaps = true and loadTexture() has completed successfully.

Parameters
lodLevelDesired minimum LOD value in [0, getMipLevels()). Clamped to the valid range internally.
