The {\ttfamily view\texorpdfstring{$<$}{<}Args...\texorpdfstring{$>$}{>}} is the primary tool for iterating over entities. It is designed to be as fast as possible by following the {\bfseries{Smallest Pool Optimization}}.\hypertarget{md__c_1_2_users_2leoam_2_c_lion_projects_2_moteur_vulkan_2_engine_2_core_2_e_c_s_2_using_01_e_c_s_autotoc_md8}{}\doxysubsection{\texorpdfstring{How it works:\+}{How it works:\+}}\label{md__c_1_2_users_2leoam_2_c_lion_projects_2_moteur_vulkan_2_engine_2_core_2_e_c_s_2_using_01_e_c_s_autotoc_md8}

\begin{DoxyEnumerate}
\item {\bfseries{Pivot Selection}}:\+ The View identifies which of your requested components is the "{}rarest"{} (the smallest pool).
\item {\bfseries{Iteration}}:\+ It iterates only over that smallest pool, skipping entities that couldn\textquotesingle{}t possibly match your criteria.
\item {\bfseries{Filtering}}:\+ For each entity in the pivot pool, it checks for the presence of the other requested components.
\end{DoxyEnumerate}

Performance Tips}}\label{md__c_1_2_users_2leoam_2_c_lion_projects_2_moteur_vulkan_2_engine_2_core_2_e_c_s_2_using_01_e_c_s_autotoc_md10}

\begin{DoxyItemize}
\item {\bfseries{Order Matters (Cache Hits)}}:\+ While the View finds the smallest pool to start with, the order of {\ttfamily Args...} in {\ttfamily view\texorpdfstring{$<$}{<}Args...\texorpdfstring{$>$}{>}} determines the order of the remaining checks. To optimize for the CPU\textquotesingle{}s short-\/circuit logic, place rarer components earlier in your template list.
\item {\bfseries{View.\+get vs Registry.\+get}}:\+
\item {\ttfamily reg.\+get\+Component\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}(e)}:\+ Performs a lookup in the Registry\textquotesingle{}s internal map/\+storage.
\item {\ttfamily view.\+get\texorpdfstring{$<$}{<}T\texorpdfstring{$>$}{>}(e)}:\+ Uses a {\bfseries{direct pointer}} to the component pool cached when the view was created. {\bfseries{Always use the view\textquotesingle{}s getter inside loops.}}
\item {\bfseries{No Branching}}:\+ The {\ttfamily for (Entity\+ID e :\+ view)} loop is a tight, predictable loop. Since the data is stored in contiguous "{}\+Dense Arrays,"{} your CPU can prefetch component data into the L1/\+\+L2 cache effectively.
\end{DoxyItemize}
