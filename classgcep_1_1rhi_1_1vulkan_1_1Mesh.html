Detailed Description

CPU-side geometry, transform, and texture data for a single 3D mesh entity.

Handles the full pipeline from OBJ loading to GPU upload readiness: vertex deduplication, AABB computation, and CPU-side buffer storage. The actual device-local buffers live in VulkanRHI's global vertex and index buffers; this class only holds staging data and metadata until clearVertices() / clearIndices() are called after the GPU upload.

Instances are stored by value inside VulkanRHI::meshes. Each instance owns exactly one VulkanTexture which may internally share GPU resources with other instances via VulkanTexture's shared_ptr-based cache.

Note
Holds a non-owning pointer to VulkanRHI — the RHI must outlive all meshes it owns.

Loads an OBJ file and optionally a texture from disk.

Parses the OBJ via ObjLoader, builds the vertex array, computes the object-space AABB, and loads the texture via VulkanTexture::loadTexture() if textureFilepath is non-empty. CPU attribute data is freed immediately after the vertex array is built. m_vertices and m_indices remain populated until VulkanRHI::uploadSingleMesh() calls clearVertices() and clearIndices().

Parameters
instanceNon-owning pointer to the owning VulkanRHI.
filepathPath to the source OBJ file.
textureFilepathPath to the albedo texture. Pass empty to skip.
transformInitial model-to-world transform (default: identity).
